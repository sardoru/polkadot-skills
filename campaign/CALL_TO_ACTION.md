**What to look for:**
- API calls that don't match current SDK versions

**Where:** [GitHub repo link — open an issue or PR]

## FAQ

**Q: Why does Polkadot need this if we already have docs.polkadot.com?**
A: Human-readable docs and agent-readable skills are different things. AI agents need opinionated, structured, machine-parseable instructions — not tutorials with multiple paths. Solana understood this; that's why they shipped a separate SKILL.md alongside their docs.

**Q: Why not just submit a PR to the Polkadot docs repo?**
A: We plan to. But maintenance costs money. AI agents break when docs go stale. The treasury proposal funds 3 months of tracked maintenance against every SDK release — not just a one-time PR.

**Q: What if another team builds a competing SKILL.md?**
A: Great. The ecosystem wins either way. But our version is built, validated, and ready today. First-mover advantage matters when AI agents are choosing which skill files to ingest.

**Q: Why the "Agent Spaces" stuff? Isn't the SKILL.md enough?**
A: The SKILL.md solves today's problem (developer onboarding). Agent Spaces solves tomorrow's problem (on-chain agent infrastructure). They're complementary but independent — Phase 1 delivers value even if Phase 2 never happens.

**Q: $10,000 for documentation?**
A: It's not documentation. It's developer acquisition infrastructure. Every AI agent that reads this file generates correct Polkadot code instead of correct Solana code. The ROI is measured in developers onboarded, not pages written.
